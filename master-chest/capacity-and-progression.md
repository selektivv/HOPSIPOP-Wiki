# Progression and Feature Unlocks

[Capacity](../capacity.md), [ranks](../ranks.md), and milestone progression unlock storage features and server benefits. Read [Capacity](../capacity.md) first for the full explanation of earning, spending, storage limits, claims, and upgrade effects.

## Checking Progress

- Run `/playtime` to see your storage limit, playtime, and first join date.
- Open the storage network and hover over Shortcuts to compare total items with free space.
- Open `/perks` to view [rank](../ranks.md) requirements and benefits.
- Open `/roadmap` to view feature-progression requirements.

Playtime is recorded and displayed, but it does not currently add [Capacity](../capacity.md) by itself.

## Earning Progress

[Capacity](../capacity.md) is awarded by HOPSIPOP activities and progression. Current sources include dungeon milestones, eligible Masterlands mob kills, mining [counter](../capacity-world/counters.md) goals, voting rewards, daily-event prizes, Vote Crate rewards, and other server rewards.

The scoreboard and reward messages show changes as they happen. Use the in-game roadmap and menus as the live source for your next goal.

## Feature Unlocks

| Progression step | Connected benefit |
| --- | --- |
| Complete Verdant | Unlock the Master Chest storage network. |
| Complete Mobmon | Unlock OmniSync crafting. |
| Complete Stonemason | Unlock Stone Cut automation. |
| Complete Crafterlands | Unlock Craft automation and progress toward Furnacity. |
| Reach Infinity wave 21 | Unlock the portable Workbench. |
| Reach Infinity wave 50 | Unlock Ender Chest access. |
| Complete Furnacity | Unlock Smelt automation. |
| Reach Atom [rank](../ranks.md) | Unlock the [Capacity](../capacity.md) World and its repeatable mining progression. |

The armor steps shown in `/roadmap` are also required parts of the progression sequence.

## Ranks

Read the dedicated [Ranks guide](../ranks.md) for the complete order, tracking controls, [rank-loss behavior](../ranks.md#how-rank-changes-work), and connected benefits. Requirements can change, so the [Rank Overview](../ranks.md#checking-and-tracking-a-rank) inside `/perks` remains the live source for current values.
